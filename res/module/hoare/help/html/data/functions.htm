Den Beweisbaum aufbauen

Regel anwenden

Sie können eine Regel anwenden, indem Sie den gewünschten Knoten markieren und in der Verifikationsregel-Anzeige eine Regel auswählen.

Nach Anwendung der Regel wird der Baum nach oben hin erweitert. Voraussetzung dafür ist allerdings, dass Sie eine Regel gewählt haben, die uach tatsächlich auf den Knoten anwendbar ist. Ist dies nicht der Fall, werden Sie über den Event-Log benachrichtigt.

Vor/Nachbedingung eingeben

Die Hauptarbeit des Nutzers liegt darin, die Vor bzw Nachbedingungen der Knoten einzugeben. Da dies nur an einigen Knoten nötig ist, wird es vom Programm übernommen. An folgenden Knoten hat der Nutzer die Möglichkeit die Bedingungen zu ändern:
  • Am Wurzelknoten (Vor- und Nachbedingung)
  • An einem "Stärkere Vorbedingung"-Knoten (Nur Vorbedingung)
  • An einem "Schwächere Nachbedingung"-Knoten (Nur Nachbedingung)
  • Am rechten Child-Knoten einer Statment-Sequence (Nur Vorbedingung)
  • Am linken Child-Knoten einer Statment-Sequence (Nur Nachbedingung)

Bei der Eingabe sind folgende mögliche Funktionen über die Toolbar implementiert:
  1. Logisches "Und"
  2. Logisches "Oder"
  3. Logisches "Nicht"
  4. Summenfunktion (Syntax: Sum(Funktion,Laufvariable,von,bis) )
  5. Produktfunktion (Syntax: Prod(Funktion,Laufvariable,von,bis) )
  6. Betragsfunktion (Syntax: abs(x) )
  7. Natürlicher Logarithmus (Syntax: ln(x) )
  8. Potenzfunktion (Syntax: x^a )
  9. Wurzelfunktion (Syntax: sqrt(x) )
  10. Kleiner-Gleich (Syntax: x1<=x2 )
  11. Größer-Gleich (Syntax: x1>=x2 )

Regel verifizieren

Nach Eingabe der Vor/Nachbedingung an Stärkeren Vorbedingungen/Schwächeren Nachbedingungen und Zuweisungsaxiomen müssen diese auf ihrer Korrektheit überprüft werden. Die Dauer dieser Überprüfung kann mit Hilfe des Schiebereglers in der Modul-Toolbar eingestellt werden. Dabei steht Ihnen eine Spanne zwischen 1 Sekunde und 1 Stunde zur Verfügung. Desweiteren wird in der Toolbar die Anzahl der noch ausstehenden Evaluierungen angezeigt.
Eine Sanduhr neben dem Knoten signalisiert, dass dieser Knoten gerade überprüft wird.
Ein grünes Häcken signalisiert, dass die Evaluation erfolgreich war.
Ein rotes Kreuz signalisiert, dass die Evaluation fehlgeschlagen ist. In diesem Fall wird eine Fehlermeldung inklusive der fehlgeschlagenen Variablenkonstelation über das Event-Log ausgegeben.

Programm komplett verifiziert

Wenn alle Zuweisungsaxiome des C0-Programmes korrekt evaluiert worden, werden diese in fetter Schrift dargestellt. Dies bedeutet, dass Sie das Programm korrekt evaluiert haben und fertig sind.
